# Glossary

## Core terms

**Project**  
A Maker Lab work unit around which users collaborate and request resources.

**Project member**  
A user assigned to a project. Members can create or participate in requisitions depending on permissions.

**Supervisor**  
A user who oversees project progress and has visibility over project activity.

**Lab technician**  
The operational user responsible for inventory, approvals, assignments, and returns.

**Equipment model**  
A product or asset type tracked by the inventory system, for example a specific board, tool, or device category.

**Equipment asset**  
A concrete physical instance of an equipment model.

**Equipment request**  
A requisition submitted by a project for one or more requested equipment models.

**Equipment usage**  
The assignment record connecting a physical asset to a project and request item over time.

**Status history**  
An audit trail of relevant state changes.

## External systems

**Snipe-IT**  
Open-source asset management platform used as the authoritative inventory system.

**University SSO**  
The institutional identity provider used for authentication.

**Legacy wiki**  
The previously used operational platform whose data and documentation inform the current redesign.

## Technical terms

**Application server**  
The backend runtime that implements business rules and system integrations.

**Reverse proxy**  
The front network component that terminates incoming HTTP(S) requests and routes them internally.

**Migration module**  
A one-time or limited-use component intended to import legacy data into the new system.
